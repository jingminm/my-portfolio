EECS 440 P1 - Beyster 1695: The Staff Maintenance Simulator

This project transforms a virtual recreation of the Beyster 1695 laboratory into a reactive "Staff Simulator" where players utilize a custom affordance-based interaction system to resolve environmental disturbances, manage NPC requests, and trigger dynamic audio-visual rewards.

In this simulation, the Beyster 1695 laboratory is populated with interactive malfunctioning computer terminals that glitch with disruptive video, noisy printers that emit sparks, and NPC characters that request the return of a physical flash drive. Beyond these core tasks, the environment includes a fire alarm that toggles emergency lighting, flickering ceiling lights, a copy machine that duplicates nearby objects, and a whiteboard that allows for persistent freehand drawing. Players navigate these challenges guided by a tutorial flier on the wall, ultimately triggering a reward sequence featuring a clapping audio cue, flashing green lights, and a spawned donut once all duties are fulfilled.

Technically, these interactions are driven by a custom Affordance Actor Component system and a global TaskCounter variable managed within the VRGameMode. The simulation utilizes LineTraceByChannel and Break Hit Result for precise interaction, while Event Tick combined with Modulo and Random Float nodes drive the automated light flickering and SkySphere Sun Height "chasing" logic. Key functions include SpawnActorFromClass for duplicating objects and rewards, SetWorldRotation paired with FindLookAtRotation for NPC head-tracking, and Toggle Visibility for state-based actors. Furthermore, Widget Components are updated via Casting to display dynamic 3D dialogue, and the RefreshMaterial() function is called to synchronize the atmospheric skybox with the shifting day-night cycle.
